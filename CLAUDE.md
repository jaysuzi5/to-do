# To-Do — Claude Code Project Guide

## Project Overview

Personal to-do list application for Jay (jaysuzi5@gmail.com) and Suzanne (jaysuziq@gmail.com).
Users manage named task lists with tasks of varying priority. Phase 2 adds Amazon Alexa skill
integration so users can say "Alexa, ask my chores to add wash the windows to Jay's list"
and tasks are added via voice command. Invocation name is "my chores".

## Stack

- Python 3.12 / Django 6.0
- uv for package management
- PostgreSQL — database `to-do` on shared k8s instance
    - External (local dev): 192.168.86.201:30004

- django-allauth 65.x — email login + Google OAuth
- Bootstrap 5 + custom CSS (static/css/site.css)
- WhiteNoise for static files
- OpenTelemetry (OTLP, wraps gunicorn in production)
- Docker multi-arch: jaysuzi5/to-do:latest

- Cloudflare tunnel: todo.jaycurtis.org

## Key Entry Points

- `config/urls.py` — root URL config
- `tasks/views.py` — all views
- `tasks/models.py` — TaskList and Task models
- `tasks/urls.py` — app URL patterns
- `config/settings.py` — all config (reads from .env)
- `config/middleware.py` — JSON request/response logging

## Architecture Notes

- Two models: `TaskList` (owner + name) → `Task` (title, priority, status, due_date)
- Users see only their own lists — all queryset filters by `owner=request.user`
- Context processor `tasks.context_processors.user_task_lists` injects `nav_task_lists`
  into every template — don't pass task lists from views, they're already in context
- `added_via_alexa=True` flag on tasks marks those added via Alexa (Phase 2)
- `ALEXA_SKILL_TOKEN` env var (optional) gates the Phase 2 API endpoint
- `Task.is_overdue` property compares `due_date < timezone.localdate()` — no DB query

## Non-Obvious Conventions

- All credentials from `.env` (local) or k8s SealedSecrets (prod) — never hardcoded
- `k8s/temp.yaml` is gitignored plain text — delete immediately after kubeseal
- Slugs on TaskList are auto-generated from `{owner.pk}-{name}` — guaranteed unique

## Files Never Edit Directly

- `staticfiles/` — generated by collectstatic
- `tasks/migrations/` — generated by makemigrations

## Deployment

```bash
# Build and push multi-arch image
docker buildx build --platform linux/amd64,linux/arm64 -t jaysuzi5/to-do:latest --push .

# First deploy (or after adding new k8s env vars)
kubectl apply -f k8s/secrets.yaml
kubectl apply -f k8s/deployment.yaml

# Subsequent deploys
kubectl rollout restart deployment to-do -n to-do
kubectl rollout status deployment/to-do -n to-do

# Superuser (first time only)
kubectl exec -n to-do -it \
  $(kubectl get pod -n to-do -l app=to-do -o name | head -1) \
  -- uv run python manage.py createsuperuser
```
